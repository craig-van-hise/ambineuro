### **Phase 1: Multi-Process Environment & Core Architecture**
**Phase Status:** ✅ Completed

* **Goal:** Establish the Electron-React frontend and the Python headless backend, bridging them via WebSockets.
* **Prompt:** "Scaffold an `electron-vite` project using React, TypeScript, and Tailwind. Create a separate `backend` directory with a Python virtual environment. The `requirements.txt` must include: `torch`, `sounddevice`, `soundfile`, `numpy`, `python-osc`, `safpy`, and `websockets`. Implement an IPC bridge where the Electron Main process spawns `engine.py` as a child process and the React frontend connects to it via a local WebSocket."
* **Test 1.1 (Environment):** Write a Python test that asserts `torch.backends.mps.is_available()` is True[cite: 81, 156]. Write a test asserting `safpy` imports without missing C-library dependencies.
* **Test 1.2 (IPC Lifecycle):** Write a script asserting that when Electron quits, the Python child process terminates gracefully. Write a WebSocket ping/pong test between React and Python.

### **Phase 2: Audio Loading & Hardware I/O**
**Phase Status:** ✅ Completed

* **Goal:** Implement the high-priority audio callback and file loading.
* **Prompt:** "Implement a `.wav` file loader and a multi-threaded ring buffer in Python to decouple the hardware audio callback from the compute blocks[cite: 68, 164]."
* **Test 2.1 (Ring Buffer):** Initialize a ring buffer of size 4096. Write 2048 samples, read 1024, and assert the read pointer advanced correctly[cite: 162, 163].
* **Test 2.2 (Callback):** Mock `sounddevice.OutputStream`. Assert that the callback consumes data from the ring buffer without under-runs[cite: 168, 170].

### **Phase 3: Dual Processing Pipeline & Rotation**
**Phase Status:** ✅ Completed

* **Goal:** Integrate the existing `DecoderGraph` routing matrix into the main audio loop.
* **Context:** I have already written the core routing logic. You will find it in `backend/decoder_graph.py`. **Do not rewrite this file.** Your job is to connect it to the audio callback in `engine.py`.
* **Prompt:** "In `engine.py`, import `DecoderGraph` from `decoder_graph.py`. Instantiate it in the `AudioEngine` class. Connect the `process_block` method so that it receives the audio from the ring buffer, processes it, and sends the output to the `sounddevice` stream. Implement the mock `a2b_model` and `spatial_math` dependencies so the graph can initialize."
* **Test 3.1 (Integration):** Write a test in `test_engine.py` that mocks `sounddevice` and passes a dummy 1024-sample block through the `AudioEngine`'s callback. Assert that the `DecoderGraph.process_block` method is called exactly once per frame.
* **Test 3.2 (Dual Routing):** In `test_decoder_graph.py`, instantiate `DecoderGraph`. Pass a block with state = 'NEURAL'; assert output is shape (Batch, 2, 1024). Pass the same block with state = 'LINEAR' and method = 'MagLS'; assert `safpy` returns the correctly shaped stereo array.

### **Phase 4: Head Tracking & State Control**
**Phase Status:** ✅ Completed

* **Goal:** Enable remote OSC control and handle UI state changes.
* **Prompt:** "Implement an `OSCController` class in Python listening for UDP packets on port 9000. Normalize incoming degrees to radians[cite: 139, 140]. Implement a WebSocket listener in `engine.py` that updates the active decoder method and loads `.sofa` files into `safpy` when paths are received."
* **Test 4.1 (OSC Mapping):** Input Yaw=90, Pitch=0, Roll=0[cite: 213]. Assert the controller outputs the correct Euler angles in Radians (e.g., [1.57, 0, 0])[cite: 214].
* **Test 4.2 (Concurrency):** Simulate the OSC thread writing angles while the Audio thread reads them[cite: 216]. Assert no race conditions[cite: 217].

### **Phase 5: Frontend UI & Visualization (via Stitch MCP)**
**Phase Status:** ✅ Completed

* **Goal:** Build the React/Tailwind interface using the connected Stitch MCP.
* **Context:** You have access to the Stitch MCP server. You must **strictly** use its provided tools to generate the React and Tailwind UI components. Do not write the visual component code from scratch manually.
* **Prompt:** "Use the Stitch MCP tools to generate a modern, dark-mode Tailwind UI for 'AmbiNeuro'. Instruct Stitch to create: 
    1. A primary drag-and-drop zone for loading `.wav` Ambisonic files.
    2. A secondary, smaller drop zone for custom `.sofa` HRTF files.
    3. A 'Decoder Engine' control panel with a dropdown to select the active decoder (A2B, SAF-MagLS, SAF-TA, SAF-LS, SAF-SPR).
    4. A visual compass widget to map head orientation (Yaw/Pitch/Roll).
    Once Stitch returns the generated components, save them into the `src/components` directory of our Electron-Vite app. Then, wire up the WebSocket connection in `App.tsx` so the dropdown and drop-zones send the correct JSON payloads to our Python backend."
* **Test 5.1 (UI Integration):** In Vitest, mount the Stitch-generated drop-zone component. Simulate a file drop event and assert that the absolute file path is extracted correctly.
* **Test 5.2 (State Control):** Simulate selecting 'SAF-MagLS' from the Stitch-generated dropdown. Assert the correct JSON payload `{"decoder": "SAF-MagLS"}` is emitted over the WebSocket.
